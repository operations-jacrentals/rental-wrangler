Judgment calls — for Jac to react to

  • The five plates ARE the wizard. Reused the exact plate grammar from the detail views instead of a separate "wizard" widget — one component vocabulary, no new pattern to learn. Steps collapse to a one-line summary once resolved (a real rep won't want Who's search box sitting open through the whole call).
  • "How Much" defaults open, not "Who." In the mid-call panel the interesting state is live-quoting — earlier steps default collapsed-done so the number is what's on screen. All five stay independently expandable by chevron.
  • Fitness gate rendered as a note + a struck-through, hover-explained row (reusing the failed Highrise · 4471-K unit from the detail views), not the literal Gate component — Gate turns a status via a picker; this is a read-only availability filter, a different job even though the brief calls it a "gate."
  • Boudreaux Marine's "owed 16d" surfaces the instant she's matched — same red Signal as her detail view, same invoice #4460 — so sizing-up costs the rep zero extra clicks. The member-rate savings ($267 vs $1,080) is offered as an upsell tip, not a Door — no money action fires itself.
  • Protection defaults OFF to match her real account state (the "protection off" stamp already on her Rental plate) — toggling it live-updates the quote, demonstrating "live" honestly, while tax stays labelled estimated since jurisdiction/final tax isn't known until invoice time.
  • Quote math is real, not invented: Skid Steer 75hp non-member, 3 days, is rate1Day×3 = $1,080 under the app's actual cheapest-blend pricing (checked against rentalPrice() in app.js); member rate would be memberDaily×3 = $267.
  • Not built: real typeahead search (the input/results here are a fixed demo set), true click-to-pick calendar range selection (quick-picks are wired; individual day clicks are decorative), and what happens if NO unit is fit to go out (an all-excluded empty state) — flagged rather than guessed.
