Rental Wrangler — Button System Draft v2 · reconciled with the app

v2 stops inventing a parallel orange ladder. Instead it fixes Primary's contrast, gives every orange treatment exactly one meaning, adds the interaction states production needs (busy · focus · reduced-motion), tokenizes the metrics, and lets money stay green. Everything below is token-driven so it ports cleanly.

What changed from v1 5-lens critique

Contrast fixed. White-on-orange was ~2.6:1 (fails AA, worse on hover). Primary now uses near-black ink on the brand orange, and hover/active step darker so legibility improves as you interact.
Orange disambiguated. Solid-orange = Primary or selected · soft-orange = armed/toggled only · orange text = branded shortcut · warm border = hover. One treatment, one meaning.
Hierarchy by elevation, not hue. Primary is raised; a selected tab/segment is flat — so a lone Primary still wins even when a tab is selected (same color, different plane).
Money stays green. Charge is green + confirm-gated; Refund is the red ladder. Never the same paint as create-Primary.
“Accent” tier dropped. Its soft-orange fill collided with the app's “armed/toggled”. ＋Rental/＋Customer re-skinned as orange-text + neutral border.
Missing states added. Busy/loading, a real :focus-visible ring, and a prefers-reduced-motion guard (the app currently ships none).
Renamed the icon tier .ghostbtn — the app already owns .iconbtn; porting as-is would've blanked every toolbar control.
Metrics tokenized. Heights 28/34/44 · radius 9/11/12, with a migration map (the app actually runs 26–44px / 8–16px today).

The one rule that makes it readable: one orange, one meaning

SOLIDSolid orange + dark ink = the one Primary action, or the selected item in a set (tab/segment). SOFTSoft-orange fill = armed / toggled-on only (a mode is active). Never a resting button. TEXTOrange text + neutral border = branded shortcut (＋Rental/＋Customer). WARMWarm border + surface lift = hover. Transient, never confused with selected.

The rules (v2)

  • One orange, one meaning. Solid = Primary/selected · soft-fill = armed/toggled · text = branded shortcut · warm border = hover. Nothing else gets orange.
  • Selected is flat; Primary is raised. Hierarchy rides on elevation, so a lone Primary wins even beside a selected tab.
  • Contrast is non-negotiable. ≥4.5:1 text, ≥3:1 fill-edge/focus, in both themes. Interaction makes contrast better, never worse.
  • Money is green, destructive is red — never Primary. And both get a real confirm (Cancel-focused, Esc-cancels, busy-on-fire).
  • Every interactive thing has a visible focus ring (:focus-visible) and respects prefers-reduced-motion.
  • Busy state on anything async — spinner, locked width, no double-fire. (This is what makes the money confirm actually safe.)
  • Icon-only ⇒ aria-label + tooltip, ≥36px (28px hover-only card controls get 44px hit-slop).
  • Three sizes, token radii. No one-off heights — it's all variables now.
  • Pills aren't buttons. Status/reference/inline-add pills keep their own system; don't drag them onto the ladder.
